1 Project Overview

VR Electric Vehicle is an immersive VR training application that teaches users how to perform battery diagnostics on electric vehicles. Designed for Meta Quest and Windows Standalone VR headsets, it walks trainees through a complete diagnostic workflow — from putting on safety equipment to interpreting OBD-II scanner data and answering condition-assessment questions.

The project was built by Ziber Art Studio using a linear, guided-procedure model: each of the six steps must be completed before advancing, with real-time task tracking, visual feedback (checkmarks, highlights, UI panels), and a final scored results clipboard.

🔄 Scene Transition System

The SceneTransitionManager singleton orchestrates scene changes with a fade-to-black effect using FadeScreen. The fade screen renders a full-screen material whose alpha is animated along an AnimationCurve, with configurable color and duration. Two modes are available:

  • Synchronous — GoToScene(index): fade out → wait → load scene
  • Asynchronous — GoToSceneAsync(index): fade out → load in background → activate when ready

🏠 Start Scene Build Index 0

The main menu scene managed by GameStartMenu. Contains three UI panels:

  • Main Menu — Start, Options, About, Quit buttons
  • Options — Volume slider (AudioListener.volume) + Turn Type dropdown (Snap vs Continuous, persisted via PlayerPrefs("turn"))
  • About — Project credits and information

🎮 Game Scene Build Index 1

The main gameplay scene containing the vehicle, battery compartment, OBD scanner, diagnostic UI, and all interactive elements. Controlled by BaterryDiag — the central gameplay controller.

4 The 6-Step Diagnostic Procedure

0
Safety Equipment
Put on gloves and safety glasses. Grabbing gloves changes hand material color; donning glasses activates a "worn" visual on the player. Both tasks must be complete before advancing.
1
Open Battery Latch
An arrow hint appears pointing to the battery latch. The OpenLatchUI panel guides the user through releasing the latch to access the battery compartment.
2
Visual Inspection
Three toggle groups: Casing, Terminal, and Leakage. Each locks after selection. Vehicle exterior togglable (Full / Half / No Exterior) via ToogleView.
3
Connect OBD Scanner
Scanner and wire appear on a table. User grabs the scanner and brings it to the OBD port (trigger zone via ColliderManager). On connection, hover model swaps to "connected" state.
4
Read Diagnostic Data
OBD screen activates showing diagnostic parameters. Reading is optional — Next button is immediately available, but individual read statuses are tracked with checkmarks.
5
Quiz & Results
Multiple-choice battery condition questions. Correct → green indicator with explanation. Incorrect → prompts review. Finish compiles all results to a clipboard summary.

6 Core Systems

📊 6.1 Task Tracking System

The TaskGroup serializable class is the backbone of progress tracking:

public class TaskGroup {
    public string TaskName;          // Display name for the step
    public bool[] TaskStatus;        // Per-task completion flags
    public Image[] TaskResultImage;  // Checkmark images for results clipboard
}

How it works: Each step has a TaskGroup with N subtasks. As the user completes subtasks (via ColliderManager triggers, Step0(), Step3(), Step4()), TaskStatus[i] is set to true. checkTask() compares completed tasks against the total — when all are done, the Next button becomes interactable.

🔊 6.2 Audio System

AudioManager is a DontDestroyOnLoad singleton that persists across scene loads. Sound objects are configured in the Inspector with name, clip, volume, pitch, and loop settings. Playback is by string name lookup:

AudioManager.instance.Play("click_sound");

UIAudio hooks Unity UI pointer events (IPointerClickHandler, IPointerEnterHandler, IPointerExitHandler) to automatically trigger named audio clips on UI interaction.

🚶 6.3 VR Locomotion

Two turn providers from XR Interaction Toolkit:

  • Snap Turn — ActionBasedSnapTurnProvider — rotates the player in fixed angular increments
  • Continuous Turn — ActionBasedContinuousTurnProvider — smooth rotation

The user's preference is stored in PlayerPrefs("turn") and applied at startup by SetTurnTypeFromPlayerPref, which enables one provider and disables the other.

🤝 6.4 Interaction System

Trigger Zones — ColliderManager uses OnTriggerEnter to detect when tagged objects enter a zone. Configurable via CollideWithTag. Supports multiple scenarios through TaskScenario branching.

Outline Highlighting — ComponentConfig toggles QuickOutline's Outline component on inspectable objects, with a companion UI hover panel that billboards toward the player's headset.

Tooltips — ToggleHover shows TextMeshPro tooltip panels on pointer hover.

3D UI Controls — XRI Examples provide knobs, sliders, levers, push buttons, joysticks, and grip buttons for diegetic VR interfaces.

9 Build & Deployment

📦 9.1 Build Scenes

The project's build settings reference two scenes in ProjectSettings/EditorBuildSettings.asset:

⚠️ Build settings reference "Backup 6" and "Backup 7" names. When changing scenes, update EditorBuildSettings.asset accordingly.

🔨 9.2 Building the Project

  1. Open the project in Unity 2022.3 LTS (or compatible version)
  2. Go to File → Build Settings
  3. Select target platform:
    • Android for Meta Quest (IL2CPP, ARM64)
    • Windows Standalone for PC VR (Mono, x86_64)
  4. Ensure both scenes are in the build list in the correct order
  5. Click Build (or Build and Run for a connected device)

There is no CI/CD pipeline — builds are made locally through the Unity Editor.
